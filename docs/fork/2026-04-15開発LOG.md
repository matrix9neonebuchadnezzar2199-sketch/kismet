## 2. ツリー表示（AP–クライアント）

### 2.1 新規モジュール

| ファイル | 役割 |
|----------|------|
| `http_data/js/kismet_ui_treeview.js` | サイドバー `tree_view`、jsPanel、**バンド区切り**／**単純ツリー** |
| `http_data/index.html` | `kismet_ui_treeview.js` を `kismet_enhanced_loader.js` 直前に読込 |
| `http_data/js/kismet_enhanced_loader.js` | `kismet_ui_treeview_module.registerSidebar()` |
| `http_data/css/kismet_enhanced.css` | `.tree-*`、信号色、後述のバンド帯・WL バッジ |

### 2.2 データ・アルゴリズム

- 取得: `POST devices/views/phy-IEEE802.11/devices.json`（`json` に `fields`、`page=0&length=50000`）
- AP: `kismet.device.base.type === "Wi-Fi AP"`
- クライアント様: `Wi-Fi Client` / `Wi-Fi Bridged`
- 親子: `dot11.device.last_bssid` を正規化して AP の `macaddr` と照合
- 未接続グループ: `last_bssid` が `00:00:00:00:00:00`／未定義、または一覧内に親 AP が無いクライアント
- バンド: `kismet.device.base.frequency` に対し閾値 5925000 / 5000000 / 2400000（kHz 想定）
- 折りたたみ: `localStorage` キー `kismet.treeview.collapse.*`
- 監視: `kismet_ui_signal_monitor.OpenSignalMonitor(device_key, mac, name, manuf)`

### 2.3 仕様変更（ユーザー要望に基づく）

| 変更 | 内容 |
|------|------|
| 更新間隔 | `REFRESH_INTERVAL` **10000 ms**（当初 15000） |
| セッション保持 | `treeDeviceCache`（MAC→最終スナップショット）にマージし、**API から消えても行を削除しない**（パネル閉鎖でキャッシュ破棄） |
| ホワイトリスト連携 | `isWhitelisted` で盾バッジ＋行クラス、`kismet-whitelist-changed` でキャッシュのみ再描画 |
| ステータス文言 | `treeview.status_poll`（直近取得件数／セッション累計） |
| バンド帯 UI | 機器行と区別する **インディゴ系リボン**（`.tree-band-header` グラデ＋左アクセント） |

### 2.4 事前検討（設計メモ）

- Kismet は `last_bssid` と `associated_client_map` で親子を推論可能（パッシブ推定である点に注意）
- ツリーはメイン Tabulator と独立取得（裏でリストは従来どおり動作）

### 2.5 i18n・テスト

## 3. Linux GUI 起動ランチャー（ターミナル不要の流れ）

| ファイル | 役割 |
|----------|------|
| `scripts/kismet-gui-launch.sh` | 非 root 時 `pkexec` で再実行 → 任意で `iw` モニターモード → `cd` clone 直下 → バックグラウンドで HTTP 待ち後 `xdg-open` → `exec kismet` |
| `scripts/kismet-gui-launch.desktop` | `Terminal=false`、`Path=..`（`.desktop` が `scripts/` 内にある前提で clone 直下を cwd）、`Exec=/bin/bash scripts/kismet-gui-launch.sh` |
| `README.md` | 上記の使い方・Windows 非対象の注記 |

**注意:** Windows 本機 Wi-Fi のモニター＋同スクリプトは想定外。Linux/Kali + polkit 前提。
