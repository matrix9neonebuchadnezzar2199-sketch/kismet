## 1. メイン取得画面（Devices タブ）× ホワイトリスト

| 項目 | 内容 |
|------|------|
| 対象 | `http_data/js/kismet.ui.js` |
| 行選択 | Tabulator 5.6 の `selectableRows: true` + `rowSelection` / `titleFormatter: rowSelection` |
| 個別登録 | 行ごとの「+」列 → `kismet_whitelist_api.addToWhitelist` |
| 一括登録 | ツールバー「選択機器をホワイトリストに登録」→ `addBulkToWhitelist` |
| 全選択 | 「このページを全選択」チェックボックス |
| 自動更新対策 | `device_key` をキーにした `Map` で選択を保持し、`replaceData` 後に再選択 |

**操作フロー（整理）**

1. Devices タブ表示 → `#center-device-extras` に信号/種別フィルタに加え、WL 用ツールバーが出る（`kismet_whitelist_api` 読込前提）。
2. 左端: [チェック] [+] [既存アイコン列] … データ列。
3. チェックで選ぶ → `Map` にエントリ蓄積 → 1 秒更新後も同じ `device_key` ならチェック復元。
4. 一括ボタン → 確認ダイアログ → 結果 `alert`（追加件数・スキップ件数）→ 選択クリア・`redraw`。
5. 行クリックでデバイス詳細は従来どおり；チェック列・「+」では開かない。

**関連**

- `http_data/locales/en|ja/translation.json` — `device_list.wl_*` キー追加
- `http_data/css/kismet_enhanced.css` — `.device-list-wl-toolbar` / `.device-list-wl-count`
- `document` の `kismet-whitelist-changed` で表の `redraw`（承認アイコン列の追随）

## 7. 本日の Q&A・判断メモ（コード外の整理）

- **Kismet キャプチャ `Operation not permitted`**: 多くは `sudo kismet` 等の権限不足。別件として記録。
- **Web UI ログイン**: 資格情報は起動ユーザーの `~/.kismet/kismet_httpd.conf`（root なら `/root/...`）。ブラウザの古い保存パスワードとズレやすい。
- **デバイス CSV エクスポート**: メタデータ一覧。Wireshark 用の**パケット**は pcapng ログ設定が別。
- **ホワイトリスト「標準で書き出せるか」**: デバイス CSV は UI から。パケット pcapng はログ設定次第。
